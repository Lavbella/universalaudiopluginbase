[EN]

Ubuntu/Linux Environment Setup Guide 

1. Install Graphical, Audio, and Core Dependencies
Open your terminal and run this command (copy and paste the entire line) 
to install the development libraries for X11, OpenGL, and ALSA:
bash
sudo apt update && sudo apt install -y libx11-dev libxext-dev libxinerama

2. Install CMake and Build Tools
Install CMake and ensure that all compiler tools are ready to use:
bash
sudo apt update && sudo apt install cmake build-essential -y

3. Install Font and System Libraries
Installing the missing font libraries and remaining source dependencies required by JUCE on Ubuntu:
bash
sudo apt update && sudo apt install -y libfreetype-dev libfontconfig1-dev

4. Install GTK and WebKit Components
bash
sudo apt update && sudo apt install -y libgtk-3-dev libwebkit2gtk-4.1-dev

Fix dependency conflicts by installing the development libraries for GTK3 and WebKit:
bash
sudo apt update && sudo apt install -y libgtk-3-dev libwebkit2gtk-4.0-dev

5. Create System Symlinks for JUCE Compatibility
Once the installation finishes, make sure to run these commands to 
create the symlinks that trick JUCE into finding the correct header paths:

sudo ln -sf /usr/lib/x86_64-linux-gnu/pkgconfig/webkit2gtk-4.1.pc /usr/lib/x86_64-linux-gnu/pkgconfig/webkit2gtk-4.0.pc
sudo ln -sf /usr/lib/x86_64-linux-gnu/pkgconfig/javascriptcoregtk-4.1.pc /usr/lib/x86_64-linux-gnu/pkgconfig/javascriptcoregtk-4.0.pc
sudo ln -sf /usr/include/libsoup-3.0/libsoup /usr/include/libsoup
sudo ln -sf /usr/include/javascriptcoregtk-4.1/jsc /usr/include/jsc 2>/dev/null || true
sudo ln -sf /usr/include/javascriptcoregtk-4.0/jsc /usr/include/jsc 2>/dev/null || true
sudo ln -sf /usr/include/webkitgtk-4.1/jsc /usr/include/jsc
sudo ln -sf /usr/include/webkitgtk-4.1 /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/webkitgtk-4.0 /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/javascriptcoregtk-4.1 /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/javascriptcoregtk-4.0 /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/webkitgtk-4.1/JavaScriptCore /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/webkitgtk-4.1/jsc/JavaScriptCore /usr/include/JavaScriptCore 2>/dev/null || true
sudo ln -sf /usr/include/webkitgtk-4.1/JavaScriptCore /usr/include/JavaScriptCore

6. Install Network Utilities and Download JUCE Source
bash# 

#1. Install Soup network development libraries
sudo apt update && apt install -y libsoup-3.0-dev

# 2. Download the official JUCE master ZIP archive
wget https://github.com

# 3. Extract the downloaded ZIP file
unzip master.zip

# 4. Rename the extracted folder to match CMake expectations
mv JUCE-master JUCE

7. VS Code Configuration and Compilation 

StepsPress Ctrl + Shift + P.
Type CMake: Select Build Variant (the word Build was added to this command name in recent updates).
Select Debug from the list that appears.
After selecting Debug mode, run the command CMake: Configure (via Ctrl + Shift + P) to update the project target.
You can now press F7 to initiate the full build process!

Troubleshooting in VS Code:

How to apply a fix if CMake throws an error cache mismatch:

Save the modified CMakeLists.txt file after pasting the updated configuration code.
Press Ctrl + Shift + P.
Type and select CMake: Delete Cache and Reconfigure (this is mandatory to flush out old configurations and 
re-evaluate paths from scratch).
